Causality is a Shiny web application I built for CS130 that makes synthetic control analysis — one of the most powerful methods in observational causal inference — accessible to researchers without requiring them to write a single line of R code. Upload your panel data, follow the guided steps, run placebo tests to validate your results, and export your findings.

the problem with causal inference

powerful method, painful to use without a coding background

The synthetic control method (Abadie, Diamond & Hainmueller) is one of the most credible tools for estimating causal effects in policy research. But running it requires knowledge of R, the Synth package, and how to interpret placebo tests. Most researchers who need it most — social scientists, policy analysts, economists without a coding background — can't access it.

Steep coding barrier

The Synth R package requires familiarity with data structures, matrix operations, and debugging code — before you can even start analysis.

Opaque validation

Placebo tests — essential for validating synthetic control results — are time-consuming to run and difficult to interpret without visualization support.

No shareable workflow

Running analysis in R scripts makes it hard to reproduce, share, or hand off to collaborators. A web app changes that — anyone can run the same analysis in a browser.

Core Challenge: How do you make a rigorous statistical method accessible to researchers who shouldn't need to know R — without sacrificing the method's integrity?

the solution — a guided Shiny app

six steps from raw data to validated causal estimates

Causality wraps the entire synthetic control workflow in a guided six-step interface. Users only need to bring their panel data — the app handles the R code, the optimization, the visualization, and the export. No terminal, no script, no debugging.

library(shiny); library(Synth); library(ggplot2)

Upload Data

Import a panel dataset (CSV). The app validates column structure and previews the data immediately.

Select Variables

Choose the outcome variable, treatment unit, control units, and pre-treatment period — guided by the UI.

Set Parameters

Define predictors, time period, and optimization settings. Defaults are provided to get started quickly.

Run Synthetic Control

The app runs the Synth optimization, constructs the synthetic control unit, and visualizes the pre/post comparison.

Run Placebo Tests

One-click placebo testing — in time and in space — to assess the statistical significance of the estimated effect.

Save & Export Results

Download plots, weight tables, and parameter settings as a reproducible record of the full analysis.

STEP 05

Run Placebo Tests — Validate Your Results

Placebo testing is the most critical step for assessing whether the estimated effect is real or a statistical artifact. The app runs three types: in-time placebos (treating pre-treatment years as if they were the treatment year), in-space placebos (running the same analysis on each control unit as if it were treated), and full placebo result inspection.

Step 5a — Run placebo plots · one-click validation launch
Step 5b — Placebo tests executing · iterating across all control units

Why placebo tests matter: A synthetic control result is only credible if the treated unit's post-treatment gap is much larger than the gaps for all the control units in the same period. Placebo tests make this comparison visual and intuitive.

STEP 06

Inspect Results & Export

The results view presents all the analysis outputs in a readable format: the main synthetic control plot, placebo distribution in time and in space, weights assigned to each donor unit, and the final results table.

visualizations — making inference visible

three views that tell the causal story

The most important contribution of the app is not the computation — it's the visualization. The three output charts work together to tell the complete causal story: what happened, why it happened (the weights), and whether it's real (the placebos).

Placebo in Time — What if the treatment had happened earlier? The gap should be small in pre-treatment "fake" periods.
Placebo in Space — Running the same analysis on every control unit. The treated unit's gap should stand out from the distribution.

Weights results — which donor units contribute most to the synthetic control, and by how much

reflection

what building causality taught me

The technical challenge here wasn't the statistics — it was the interaction design. Synthetic control analysis has many interdependent parameters, and the order in which users configure them matters enormously. Getting that sequencing right, while hiding all the complexity underneath a clean UI, was the real design problem.

LESSON 01

Sequencing is UX design

In a complex analytical workflow, the order of steps is a UX decision, not just a technical one. Getting it wrong means users configure things they don't understand yet.

LESSON 02

Defaults are design

The best way to help non-expert users isn't to remove options — it's to provide sensible defaults that let them run a valid analysis first and customize later.

LESSON 03

Visualization is the method

In causal inference, a chart isn't decoration — it IS the result. The visual output of a placebo test communicates something that a p-value alone never could.
