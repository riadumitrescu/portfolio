The Research

Powerful scheduling.
Leaky trial funnel.

Acuity's feature set rivals any competitor, but the trial-to-paid journey has critical friction points that lose prospective customers at key conversion moments.

No Free Tier, Passive Trial

Unlike Calendly's freemium model, Acuity offers no free plan. The trial expires silently with no strong conversion prompts, and users can't cancel early — creating friction, not urgency.

Source: GetApp Reviews, SMB Guide Comparison

Confusing Client Booking Page

"The customer experience is confusing and many make mistakes because of it." Clients encounter slow loading, unclear navigation, and a booking flow that doesn't match the business's brand.

Source: Trustpilot, January 2026

Broken Mobile Experience

The mobile app forces 24-hour scrolling, removes the week view option, and lacks a refresh button — causing missed appointments and frustration for on-the-go business owners.

Source: App Store Reviews, JustUseApp

Complex Setup, No Guidance

Setting availability is confusing. Users accidentally leave calendars open, struggle with time zones, and some hire freelancers just to configure basic scheduling rules.

Source: Capterra, Squarespace Forum

Acquisition Identity Crisis

Dual branding (Squarespace Scheduling vs. Acuity), forced re-logins every 30 minutes, and removed features have left long-time users saying the product is "rapidly going downhill."

Source: PissedConsumer, Trustpilot

Value Not Demonstrated

Trial users configure settings but never experience the "aha moment" of a real booking. Without seeing tangible value, they let the trial expire rather than converting to paid.

Source: User Research Synthesis

The Solution

Meet Momentum

A milestone-based trial experience that guides new users to their first booking, celebrates their progress, and demonstrates real value before asking them to pay.

Before & After

From passive trial to active conversion

Without Momentum
Trial expires silently — no urgency, no conversion prompts, no value recap
Setup feels like homework — pages of settings with no guidance on what matters first
Users never reach the "aha" moment — they configure but never experience a real booking
No value quantification — users don't see the time or money Acuity saves them
With Momentum
Milestone celebrations build emotional investment and show progress toward real outcomes
Guided setup with clear priority order — complete the most impactful steps first
First booking is the conversion catalyst — users see the magic of automated scheduling
Real-time value metrics — "You saved 3 hours this week" shown at the upgrade prompt

Implementation

Roadmap

A phased approach aligned with the Growth team's experimentation model — ship, measure, iterate.

Phase 1

Milestone Framework

Design the core milestone system, define the 5 key conversion moments, and build the progress tracking UI. A/B test against the current onboarding flow.

Weeks 1–3
Phase 2

Value Demonstrations

Add real-time value metrics — time saved, bookings automated, revenue collected. Show these at key moments: milestone completion, trial midpoint, and upgrade prompt.

Weeks 4–6
Phase 3

Contextual Nudges

Design trigger-based prompts at critical moments: first booking celebration, 48-hour trial warning, and personalized upgrade recommendations based on usage patterns.

Weeks 7–9
Phase 4

Experiment & Optimize

Run multivariate tests on milestone order, value messaging, and upgrade timing. Analyze funnel data to identify and fix remaining drop-off points.

Weeks 10–12

About the Designer

Maria Dumitrescu

I'm a design student passionate about growth-focused product experiences. Acuity Momentum represents my vision for how the trial-to-paid journey should feel: not like a countdown to a paywall, but like a guided path to real value that makes the upgrade decision obvious.

This proposal is grounded in real user feedback from Capterra, Trustpilot, G2, GetApp, and Squarespace community forums, and aligns directly with the Acuity Growth team's focus on experimentation and reducing friction.
