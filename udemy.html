Ideas

AI fluency for the design team

Starting ideas, not a final plan. I'd shape everything with you and the team.

01 · Learn together

Hands-on AI sessions

Recurring sessions where the team learns AI tools together. Not slides. Hands-on with real design tasks. Format that works best: show a technique, let people try it right away, then next thing.

What is MCP and why should designers care?

MCP (Model Context Protocol) is like USB-C for AI tools. One standard plug so Claude can talk to Figma, GitHub, and other tools through the same protocol. For designers: point Claude at your Figma file and it reads your components, sets variables, generates code. I'd demo this live and let people try it.

Figma Make: prompts to prototypes

Drop a design file in and chat with AI to build a working prototype. As of April 2026, Make Kits let you import your design system's tokens and code packages so everything stays on brand. If Udemy's system gets set up as a Make Kit, every prototype would automatically follow it.

Claude Code to Figma: code back to canvas

Launched Feb 2026. Build something in Claude Code, capture the live UI, send it into Figma as editable frames. Actual layers and components, not screenshots. Design and code flow both directions.

Vibe coding from zero

Setup is where beginners get stuck. What is a terminal. What is GitHub. From my workshops, once people get past setup they're excited. But if they can't, they give up. Patient step-by-step guide, first session together.

AI for everyday design work

No code needed. Writing prototype content, summarizing research, generating copy variations, drafting docs. Low-barrier entry for people not ready for vibe coding yet.

On the senior designers: the discomfort isn't the learning. It's being a beginner again in front of people. Every session should feel like "we're figuring this out together." Celebrating questions matters as much as celebrating what people build.

02 · Write it down

Docs that actually get used

Everything learned gets captured so it doesn't disappear after the session and new hires can onboard with it later.

→ AI setup guide (terminal, GitHub, Claude Code, Figma MCP, zero to connected)

→ MCP explainer for designers (plain language, never-read-a-spec friendly)

→ Figma file standards (naming, structure, components, archiving)

→ AI tool catalog (tools by use case with tips)

→ Vibe coding playbook (workflows, troubleshooting, team examples)

From freelance work I learned: documentation that can't be maintained isn't documentation. Everything I build would be designed to work without me.

03 · Build community

A place to be stuck together

You mentioned wanting a community where people share what they're making and where they're stuck. I love that framing because it normalizes not knowing.

→ Weekly show-and-tell. 30 min. Share what you tried, what worked, what confused you. Recorded for async.

→ Slack channel. Persistent space for tips, questions, interesting outputs between sessions.

→ Buddy pairings. Someone comfortable with AI + someone starting out. Just a person you can DM.

From my Figma workshops: events are sparks, communities are fires. The workshops were 2 hours. The Telegram community I built after is still growing.
